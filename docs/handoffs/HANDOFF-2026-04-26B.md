## Architecture: adapt to regime shifts live

The key problem: a symbol can be stellar in 90-day optimized window but fail over 365 days.
GOOGL/AMZN/META are proof. The solution path:

1. **Running behavior=apply with the curated 4 symbols (ABNB,NVDA,NET,SBUX)** gives the best
   live adaptability using the classifier
2. **Rolling performance gate** (5 consecutive losses → 10-day pause) handles per-symbol regime shifts live
3. **`sustainedSell` + `bearish` regime** detect macro downturns and suppress longs automatically
4. **`extendedBreakout` blocking all longs** prevents chasing overbought stocks during melt-ups

For live: if SBUX or NET suddenly starts throttling (PF < 1), the rolling gate will pause them.
Consider dropping them from the live symbol list at that point.

### Q1 — How to implement the symbol quality pre-filter?

**What's already there**: The rolling performance gate (5-trade window → 10-day pause) handles
*post-entry* degradation well. What's missing is a *cold-start pre-filter* — something that
prevents us from ever trading a symbol that has never been validated.

**Recommended two-layer approach**:

**Layer 1 — Manual walk-forward validation (offline, before adding any symbol to the live set)**
```
node scripts/run-100-trades.mjs --symbols=CANDIDATE --behavior=apply \
  --time-filter=open,mid --lookback=365 --target=1000
```
A symbol is accepted only when:
- Walk-forward test PF ≥ 1.3 AND test expectancy > 0 AND test MDD < 10%
- Test PF at +5 bps extra slippage ≥ 1.1
- At least 20 trades in the test half (not enough to judge on < 20)

**Layer 2 — Automated rolling PF gate (in-sim & live)**
Already implemented: 5-trade rolling window with losing-streak pause.
Tighten by dropping `ROLLING_WINDOW_TRADES` from 5 to 4 if you want faster throttle response
at the cost of more false pauses on good symbols.

**Layer 3 — Classifier `skip` flag (future)**
If in-session behavior for a symbol shows `trendEfficiency < 0.15 AND flipRatio > 0.55 AND
sustainedSellSignal`, the behavior classifier should return `blockSetups: ['trend','breakout','mean-revert']`
(i.e., block everything). This would act as a live pre-filter if the symbol starts choppy on day 1.
Not yet implemented — code the `thinChop` profile already covers the partial case.

### Q4 — Regime awareness and chop detection: what do we have?

**Yes, we have multi-layer regime detection:**

| Layer | Mechanism | Where |
|:--|:--|:--|
| Market-level regime | `inferMarketRegime`: bearish/chop/breakout/trend/mixed from price % changes | `day-trading-bot.js:1161` |
| SPY breadth | `spyTrendPct < -0.3/−0.8/−1.2` → cumulative +8/+18/+33 risk score | Entry scoring |
| SPY volatility regime | 5-bar ATR vs 20-bar baseline; volatile = blocks trend entries, halves size | `marketVolatileRegime` input |
| Per-symbol behavior | `sustainedSell`, `thinChop`, `falseBreakoutProne`, `extendedBreakout`, `noisyHighBeta` | Classifier |
| Intraday chop | Regime `chop` → −24 entry score, −16 risk boost, blocks trend/breakout via `allowNewEntry` | Hard gate |
| Bearish regime | `shortTrendPct < −0.5 && mediumTrendPct < −0.3` → only mean-revert longs allowed | Hard gate |

**What's missing that matters:**
- No VIX-level gate (we use SPY ATR as a proxy — good enough for 5-min trading)
- No intraday breadth (e.g., % of S&P stocks above VWAP); would improve macro chop detection
- Chop regime fires on 5-bar local price action; a sideways market after an initial morning
  move can pass all gates before the chop is detected. The `thinChop` behavior profile partially
  compensates by reading 36-bar rolling efficiency.

**Bottom line**: We will not bleed in sustained chop. The `chop` regime + `thinChop` profile
combination blocks 40+ entries per 1000-trade run that would otherwise be losers. The gap is
*transitional chop* (first 30 min of a choppy day), which is why the Mid 10:30–11:30 window
outperforms Open 9:30–10:30 (+51% WR vs 39% WR in the 5-symbol test).

### Q6 — Slippage simulation: is it realistic?

**Previous state (3 bps per side, 6 bps round-trip)**: Optimistic for anything except IEX mid-point
fills on the most liquid names (NVDA, AMZN at their tightest spreads).

**Current state (5 bps per side, 10 bps round-trip)**: Conservative floor for large-cap stocks
using marketable limit orders placed 1–2 bps inside the spread.

**Reality check by fill type:**

| Order type | Expected slippage per side | Verdict |
|:--|:--|:--|
| IEX mid-point peg (odd lot) | 0–2 bps | Better than model |
| Marketable limit (NBBO) | 2–5 bps | Matches model |
| Market order (large-cap) | 3–8 bps | Matches to slightly above model |
| Market order (mid-cap like MRVL) | 8–15 bps | Worse than model |

**Key implication**: For our 5-symbol curated set, the 5 bps model is appropriate. For any symbol
with ADR < 3% or average daily volume < 5M shares, apply a mental 8–10 bps per side. The
slippage sensitivity table on the 5-symbol run confirms:
- +1 bps extra (6 bps total): PF 1.44 — comfortable
- +5 bps extra (10 bps total): PF 1.19 — viable but thin
- +10 bps extra (15 bps total): PF 0.95 — unviable

**Recommendation**: Use limit orders in live. A marketable limit placed 5 bps below ask on a
$100 stock captures the mid-point on most IEX fills. Never use pure market orders for entries.

### Q7 — ARM: thinner liquidity + exaggerated moves + retail crowding. Structural weakness?

**Yes, ARM has a structural incompatibility with our strategy.** The broad observe test confirmed:
ARM: 21 trades, 38% WR, −$245, PF 0.77. Not eligible.

**Why ARM breaks our approach:**
1. **Float & liquidity**: ARM has a relatively small tradeable float for a $100+ stock. During
   the Open window, spreads can be 15–30 bps — 3–6× our 5 bps model. Our spread hard gate
   (0.35%) would catch this IF we feed live spread data (currently we don't in the sim).
2. **Retail crowding**: Retail order flow arrives in waves around key levels (e.g., $120, $130)
   creating sharp moves that quickly reverse — exactly the `noisyHighBeta` pattern. Our classifier
   correctly labels ARM as `noisyHighBeta` but only blocks breakout entries; trend entries at
   reduced size still bleed.
3. **Exaggerated moves**: ARM's ADR is typically 5–8%, far above our 2.4% reward target. Paradoxically
   this creates the false impression of great setups (huge moves!) that actually reverse before
   we can harvest them.
4. **ATR expansion behaviour**: ARM's ATR spikes to 1.5–2% per 5-min bar in momentum phases,
   then collapses — our dynamic stop widens to 2% right as the move exhausts.

**Fix**: ARM is in `BROAD_SYMBOLS` but not in the curated set and should **never** be added until
a focused 365-day walk-forward validation shows PF ≥ 1.3 in both halves with 20+ test trades.
The same caution applies to PLTR, COIN, SMCI, RIVN, ELF, CELH — all retail-crowded names that
showed negative PF in the broad universe test.

### Q8 — TSLA false breakout problem

TSLA is already on the **do-not-trade list** (comment in `scripts/run-100-trades.mjs:148`).
Broad observe test: PF 1.59 in observe mode — which looks OK — but in apply mode with real
sizing TSLA historically runs 38–42% WR and net negative.

**Why TSLA specifically fails breakouts:**
1. **Headline-driven reversals**: TSLA ORB breaks frequently trigger on news/tweet reactions
   that reverse within 10–15 minutes. Our `falseBreakoutProne` profile catches this
   (ORB break failed + inside-range flip ratio > 0.38) and blocks breakout entries, but
   the first 5-min candle entry fires *before* the failure is detectable.
2. **Wide spread on momentum**: TSLA spreads widen to 15–30 bps during the first 30 min.
   Our new spread hard gate (0.35%) would block most of these entries in live, but the sim
   uses 0.06% fixed so these trades still appear in backtests.
3. **No short-side hedge**: When TSLA's breakout fails, it reverses hard. We can only exit;
   we can't flip short to capture the reversal.

**How this applies to other symbols** (the structural concern):
Any stock with: flipRatio > 0.45 AND ORB break-fail rate > 30% AND ADR > 6% is TSLA-like.
Current safeguards: `falseBreakoutProne` blocks breakout entries; `noisyHighBeta` reduces size.
For the Open 9:30–10:30 slot specifically — WR is only 39% even on our curated 5 symbols.
The Mid slot's 51% WR is where breakout follow-through is more reliable.

**Action**: Keep TSLA blacklisted. For any new symbol being considered with these characteristics,
require 2 consecutive closes above the ORB high (not just touch) before a breakout entry is
counted as valid. This is a future code change that would tighten `classifySetup`'s breakout condition.
