## Files changed

### `api/day-trading-bot.js` — strategy engine
All logic lives in `evaluateDayTradingDecision(input)`.

**RISK_PROFILES changes:**
- `balanced.buyThreshold`: 52 → **55** (filters marginal entries)
- `balanced.stopLossPct`: 1.7% → **1.2%** (tighter stop)
- `balanced.trailingStopPct`: 1.2% → **0.8%**
- `balanced.rewardTargetPct`: 1.5% → **2.4%** (was sub-1R — fixed to 2R)

**New inputs accepted:**
- `nowMs` — backtest/simulation candle time (prevents timeout using real wall clock)
- `vwap` — intraday VWAP from caller
- `highWatermarkPrice` — highest close since entry (drives trailing + break-even stops)
- `dailyLossesPct` — session P&L% from caller; also auto-injected server-side
- `adrPct` — 5-day avg daily range % (blocks entries on tight-range days)
- `spyTrendPct` — SPY 5-bar trend (adds risk when market is weak)

**New indicators:**
- RSI-14, EMA-9/21 crossover, MACD (EMA12−EMA26)
- RSI > 80 hard-blocks new **trend** entries (breakout exempt)

**New scoring adjustments:**
- Afternoon penalty: −6 pts entry after 2:00 PM ET (18:00 UTC)
- ADR filter: −8 pts entry if daily range < 1.2× reward target
- SPY breadth: +8 risk if SPY trend < −0.3%; +16 risk if < −0.8%
- VWAP: +5/−8 entry; +6 risk below VWAP

**New exit logic (in priority order, inside the position-open branch):**
1. Hard stop-loss (unchanged)
2. Risk-elevated + trend-broken exit (unchanged)
3. 2R profit trim (unchanged)
4. **1R partial exit** — trim when at 1× stop with fading momentum (`shortTrendPct < 0.1`)
5. Scale-up add (unchanged)
6. **Trailing stop** — fires when highWatermark > entry×(1+trailingStopPct) and price retreats
7. **Break-even stop** — fires when highWatermark ≥ entry×(1+stopLossPct) AND price ≤ entry
8. **Stagnation exit** — fires when position flat (|PnL| < 0.35%) after 60 min
9. EOD force-close at 19:50 UTC
10. Hard timeout after 20 candles / 2 h

**New entry gates (in priority order, inside the no-position branch):**
1. Daily loss limit
2. **Market session gate** — no new entries outside 13:30–19:30 UTC
3. **RSI > 80 block for trend** — wait for RSI to cool
4. entryScore ≥ buyThreshold (existing)

### `scripts/run-100-trades.mjs` — simulation script
- Pre-fetches SPY and builds `spyTrendMap: Map<timestamp, shortTrendPct>`
- Builds `adrMap: Map<date, adrPct>` per symbol using completed prior days' H-L/C
- Passes `adrPct`, `spyTrendPct`, `highWatermarkPrice`, `nowMs`, `vwap`, `dailyLossesPct` to every decision call
- `highWatermark` resets on position close; updates to `Math.max(highWatermark, c.high)` while open
- Slippage: 0.03 bps each way on market fills
- `deriveExitReason` handles: `trailing-stop`, `breakeven-stop`, `stagnation`, `eod-close`, `stop-loss`, `timeout`, `trim-profit-target`, `risk-exit`, `signal-exit`

Run: `node scripts/run-100-trades.mjs`

## Key architectural notes

- `evaluateDayTradingDecision` is **stateless** — all state (position, highWatermark, dailyLosses) must be passed by the caller every call.
- The `nowMs` input is critical for backtesting accuracy — without it, `Date.now()` makes position timeout fire immediately on historical candles.
- SPY is fetched twice in the simulation (once for breadth map, once as a symbol). This is intentional and harmless.
- GOOGL produces 71%+ win rate in simulation and should remain in the symbol list.
- AAPL and META are volatile performers — watch them in paper trading.
