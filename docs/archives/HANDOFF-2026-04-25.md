## What was implemented

**1. Volatility-regime switch** (`api/day-trading-bot.js` + `scripts/run-100-trades.mjs`)
- `buildVolatilityRegimeMap(spyCandles)` computes per-bar volatile/calm from SPY 5-bar vs 20-bar ATR ratio
- Gate fires when `setup === 'trend' && marketVolatileRegime` — blocks trend entries in high-vol periods
- Impact: trend setup improved from ~37-43% WR (Jan-Feb) toward ~40% with smaller losses

**2. Symbol expansion** (ARM kept, PLTR/CRWD/COIN rejected)

**3. VWAP reclaim signal** (`api/day-trading-bot.js`)

- `if (vwapReclaim && setup !== 'mean-revert') entryScore += 10`
- Computed in sim: `prevVwapDevPct < -0.1 && vwapDevPct > 0.1`

**4. Per-setup exit tuning** (`api/day-trading-bot.js`)
- `const entrySetup = input.entrySetup ?? null` — stored at buy, passed back on each bar
- 1R partial trim now skips breakout entries: `&& entrySetup !== 'breakout'`
- Impact: avg win up from $94 → $112 (breakout winners held to 2R or trailing stop)

### Suggestions for next session

**A. Tighten trailing stop from 0.8% → 0.6%** (not yet tried)
- Hypothesis: winning trades that peak near 1.8-2R and pull back 0.8% exit at ~1.0-1.2R; a 0.6% trail would exit at 1.2-1.4R — better profit capture
- Risk: might exit some trades prematurely on shallow dips before they continue to 2R
- Verdict: worth a test, should add ~$100-200 to gross profit

**B. Break-even stop: tighten the 1R requirement**
- Currently: `highWatermark >= avgEntry × (1 + dynamicStop/100)` — must reach full 1R before stop moves to breakeven
- Proposal: move breakeven stop to trigger at 0.8R instead of 1R — lock in more near-breakeven exits on winners that pull back from 0.8-1.0R
- Currently 2 break-even stop exits (very rare); making it fire earlier could add 5-10 more per 90d

**C. Re-test breakout entry with lowered score threshold**
- Breakout WR is 52-54% — the durable edge (63-73% in trending months)
- The setup requires strict structural conditions already, but the score threshold applies too
- Try `buyThreshold - 5` for breakout (50 for balanced) to generate more breakout entries without relaxing structural criteria

**D. Reduce trailing stop pct specifically for trend entries**
- Breakout: keep 0.8% trailing (let it run to 2R)
- Trend: use 0.5% trailing (exit trend winners sooner once profitable — consistent with their lower continuation rate)
- Would require passing `entrySetup` into trailing stop logic
